# Purpose, scratchpad, data input, set-up process

## Learning Objectives

1.  **022.11.04.01.01** -- Describe the purpose of a CDU/MCDU.
2.  **022.11.04.01.02** -- Describe the typical layout of a CDU/MCDU and the
    general purpose of the following: screen; line select keys; menu select
    keys; alphanumerical keys.
3.  **022.11.04.01.03** -- Explain the function of the "scratchpad" part of the
    screen.
4.  **022.11.04.01.04** -- Describe how input of some data is compulsory for the
    function of the FMS and other data is optional, and that different symbology
    is used to highlight this: rectangular boxes = compulsory information;
    dashed line = optional information.
5.  **022.11.04.01.05** -- Describe a typical FMS pre-flight set-up process
    through the CDU/MCDU to cover the most basic information (with the aim to
    create awareness of required information as this is irrespective of aircraft
    type and FMS/FMGS make): ident page (who am I = aircraft type/variant,
    engine type/rating and appropriate navigation database); position
    initialisation (where am I = position for aligning the IRS and FMS
    position); route initialisation (where am I going to = place of
    departure/destination and alternate(s)); route programming (how will I get
    there = SIDs, STARS, route (company or otherwise)); performance
    initialisation (when will I arrive = weights, flap setting, FLEX/assumed
    temperature/derate, take-off speeds).
